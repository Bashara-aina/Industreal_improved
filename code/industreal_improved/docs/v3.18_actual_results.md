# v3.18 — REAL improvements at 1000-frame apples-to-apples

After YOLOv8 init + balanced LR + 17000 batches of training:

| Head | v3.7 baseline (1000f) | v3.18 b17000 (1000f) | Δ |
|------|----------------------|---------------------|---|
| Activity Top-1 | 0.37 | 0.41 | **+11%** ✓ |
| Pose MAE | 19.41° | 19.92° | ≈ same |
| PSR F1 | 0.40 | 0.44 | **+10%** ✓ |
| Detection mAP | 0.026 | 0.056 | **+115%** ✓ |

## Key findings

**The previous 500-frame PSR eval was misleading.** With more frames:
- PSR baseline is actually 0.40, not 0.17
- This is because PSR F1 is averaged over components with non-zero GT
- 500 frames only captures ~6 of 11 components
- 1000 frames captures more, giving better estimate

## What worked

1. **YOLOv8 head init**: Per-class bias calibration from GT frequencies
   - Activity: 0.42 at init (immediately +13%)
   - Detection: better-calibrated cls head → less spurious predictions
   
2. **Balanced per-head LR** (pose=10x, act/psr=50x, det=100x)
   - All heads get enough gradient signal
   - Pose LR=10x (not 50x) prevents oscillation
   
3. **Long training** (17000+ batches)
   - Detection mAP more than doubled: 0.026 → 0.056
   - Activity steadily improving
   - PSR steadily improving
