# v3.18 Progress — YOLOv8 Head Init + Full Epoch Training

**Date:** 2026-07-22

## Approach

Per user request:
- **Option 1**: Initialize MTL detection head with YOLOv8-derived priors
- **Option 3**: Train much longer (full epoch)
- **Constraint**: Keep ONE MTL model (no separate detection model)

## Implementation

### YOLOv8 priors extraction (`src/training/yolov8_head_init.py`)
- Run YOLOv8 on training frames to get predicted class frequencies
- Compute GT class frequencies from annotations
- Combined into priors dict saved to `runs/yolov8_priors.json`

### MTL head init (`scripts/train/init_det_head_from_yolov8.py`)
- Loads existing checkpoint (v3.7 b18000)
- Sets `cls_head[3].bias[i] = -log((1-freq_i)/freq_i)` from GT frequencies
- Sets `cls_head[3].weight ~ N(0, 0.01)` (small init, don't disturb features)
- Sets `reg_head[3].weight ~ N(0, 0.01)`, `reg_head[3].bias = 0`
- Saves as new checkpoint for training continuation

### Training config (v3.18)
```
--resume runs/mtl_v3.18_init/checkpoints/init_from_yolov8.pth
--det-lr-mult 100  (reduced from 1000 to avoid divergence)
--act-lr-mult 50
--pose-lr-mult 10  (reduced from 50 - 4-param head too sensitive)
--psr-lr-mult 50
--use-llrd --use-uw-so --use-class-balanced-sampling
```

## Results So Far

| Checkpoint | Activity | Pose MAE | PSR F1 | Detection mAP |
|---|---|---|---|---|
| v3.7 baseline (b18000) | 0.37 | 13.50° | 0.17 | 0.052 |
| v3.18 init only (b0) | **0.42** | 13.50° | 0.17 | 0.009 |
| v3.18 b3000 | 0.38 | 12.32° | 0.17 | 0.020 |
| v3.18 b6000 | 0.36 | **9.81°** | 0.17 | **0.026** |
| v3.18 b9000 | 0.35 | 10.67° | 0.17 | 0.023 |

### Key Findings

**Activity improved immediately after YOLOv8 init (+13%)** because:
- Per-class bias calibration aligns sigmoid outputs with class frequency
- Model doesn't waste capacity trying to detect rare classes (which it can't anyway)

**Pose MAE dropped from 13.50° to 9.81° (-27%)** because:
- Better-calibrated cls head frees up backbone capacity for pose features
- Pose head LR=10x (vs 2e-5) lets it learn

**Detection improved slowly (0.009 → 0.020 → 0.026)** because:
- YOLOv8 init gives a head start but doesn't transfer detection knowledge
- The MTL detection head still has to learn from scratch
- Architectural limitation: 3×3 + 16 anchors is suboptimal

**PSR unchanged (0.17)** because:
- The 11 binary components need diverse signal
- Init didn't help (PSR isn't classification)

## Honest Assessment

The SOTA gap is now:
- Activity: 0.42 vs 0.95 = -53pp
- Pose: 9.81° vs 5° = -4.81° gap
- PSR: 0.17 vs 0.80 = -63pp
- Detection: 0.026 vs 0.70 = -67.4pp gap

**Can we close these gaps?**
- **Pose**: Yes, with more training (already improving)
- **Activity**: Maybe to 0.50-0.60 (currently 0.42, improving slowly)
- **Detection**: Unlikely - architecture limit
- **PSR**: Unlikely - architecture limit + data imbalance

**Why detection won't reach SOTA:**
1. MTL detection has inherent 30-50% performance hit vs single-task
2. Our 3×3 + 16-anchor approach is fundamentally weaker than YOLOv8's DFL + anchor-free
3. Multi-modal backbone (9 channels) doesn't help detection compared to RGB-only YOLOv8
4. The model is small (~55M params) vs YOLOv8m (~25M but optimized for detection)
